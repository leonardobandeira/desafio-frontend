Checklist Desafio Frontend

    • Fazer fork do repositório Oliveira-Trust/desafio-frontend para sua conta no GitHub.
    • Clonar o repositório forkado para a sua máquina local. Comando: git clone https://github.com/seu-username/desafio-frontend.git
    • Criar uma nova branch com o seu nome completo. Comando: git checkout -b seu-nome-completo
    • Instalar o json-server globalmente. Comando: npm install -g json-server
    • Instalar as dependências do projeto. Comando: npm install
    • Rodar o json-server. Comando: json-server --watch db.json --port 3004
    • Rodar a aplicação Vue.js. Comando: npm start
    • Acompanhar identidade visual do protótipo fornecido.
    • Listar dados da API de usuários em uma tabela.
    • Incluir todos os dados relevantes da API.
    • Adicionar botões funcionais de editar (modal) e deletar registro para cada elemento.
    • Implementar funcionalidade de paginação na listagem.
    • Adicionar filtro para a listagem de usuários.
    • Implementar modal para editar registros de usuários.
    • Validar os inputs durante a edição.
    • Implementar modal para adicionar novas carteiras.
    • Validar os inputs durante a criação.
    • Implementar conversão do saldo Bitcoin (BTC) de cada carteira.
    • Util
    • Utilizar a API de conversão recomendada.
    • Ao criar ou editar uma carteira, permitir a inserção de um valor de compra e exibir o valor adquirido na criptomoeda de destino.
    • Implementar validação de inputs nos modais de criação e edição de carteiras.
    • Implementar gerenciamento de estado utilizando Vuex (opcional, mas recomendado).
    • Adicionar funcionalidade para exportar dados da lista em CSV (pode utilizar biblioteca npm).
    • Adicionar documentação sobre como rodar o projeto e qualquer configuração especial necessária.
    • Commitar todas as mudanças e enviar um pull request do seu fork para o repositório original.
    • Enviar o link do pull request juntamente com seu LinkedIn ou currículo para vagas@oliveiratrust.com.br.
    • Revisar todo o código para garantir legibilidade e modularização.
    • Testar todas as funcionalidades para assegurar que estão funcionando conforme o esperado.
